# 《今天干点啥》第一阶段开发执行清单

更新时间：2026-03-23

本文档用于约束当前这一轮代码改造顺序，目标是把小程序从“多页面原型”升级为“有主线、可联动、可沉淀”的第一阶段可运行版本。

## 一、总体目标

本阶段只做最关键的主线闭环：

1. 首页从入口页升级为 `Today OS` 今日面板
2. 恋人系统从本地 mock 升级为真实关系系统基础版
3. `今天吃点啥` 从随机结果页升级为 `推荐 + 发起 + 回应 + 记录`
4. 发布/发现与主线打通，支持模板化生活记录
5. 后端补齐关系、动作、记忆、仪表盘能力

## 二、模块顺序

### 模块 1：工程基础整理

目标：统一请求、用户缓存、文案和页面主色调，为后续重构打底。

步骤：

1. 检查并修正 `config/api.js`、`utils/api.js` 的基础请求能力
2. 梳理本地缓存结构，统一 `token`、`user`、`relation`、`dashboard` 的存储键
3. 为首页、吃模块、关系页定义统一的假数据兜底结构
4. 统一高频文案，去掉页面中的乱码和旧占位文案
5. 确认新增功能不破坏现有发现页、评论页、登录页

### 模块 2：关系系统后端基础版

目标：把“恋人系统”从本地存储升级为后端真实数据模型。

步骤：

1. 新增 `Relation` 模型
2. 新增 `RelationAction` 模型
3. 新增 `MemoryRecord` 模型
4. 设计关系状态字段：
   - `pending`
   - `active`
   - `paused`
   - `closed`
5. 设计关系类型字段：
   - `lover`
   - `buddy`
   - `bestie`
6. 设计关系偏好字段：
   - 昵称
   - 纪念日
   - 忌口
   - 常用地址
   - 常用时间偏好
7. 设计动作类型字段：
   - `eat_share`
   - `eat_treat`
   - `date_invite`
   - `photo_task`
   - `daily_task`
8. 设计动作状态字段：
   - `pending`
   - `accepted`
   - `declined`
   - `done`
   - `expired`
9. 在用户模型中补充轻量关系摘要字段，便于首页读取
10. 新增关系路由文件
11. 实现创建邀请接口
12. 实现接受邀请接口
13. 实现拒绝邀请接口
14. 实现解除关系接口
15. 实现查询当前关系接口
16. 实现创建关系动作接口
17. 实现回应关系动作接口
18. 实现完成关系动作接口
19. 实现获取关系时间线接口
20. 实现获取今日仪表盘接口
21. 给所有关系接口补充基础校验和默认兜底

### 模块 3：首页 Today OS 重构

目标：首页不再只是功能宫格，而是用户当天真正会使用的“生活面板”。

步骤：

1. 重构首页数据结构
2. 设计首页主视觉：
   - 温馨
   - 柔和暖光
   - 轻奶油感
   - 适合长期浏览
3. 新增“今日问候”区
4. 新增“今日心情”区
5. 新增“关系状态卡”
6. 新增“今日推荐卡组”
7. 新增 `吃 / 玩 / 拍` 三大主入口
8. 新增“待回应”与“待完成”任务卡
9. 新增“最近回忆”入口
10. 接入首页仪表盘接口
11. 无关系时展示引导态
12. 有关系时展示真实关系摘要
13. 首页支持下拉刷新
14. 首页保留向旧页面的安全跳转

### 模块 4：今天吃点啥升级

目标：把当前页面从“随机美食 + 菜谱”升级为第一阶段主引擎。

步骤：

1. 重构吃模块页面布局
2. 保留随机推荐能力
3. 新增推荐理由展示
4. 新增场景标签展示
5. 新增预算、口味、关系场景的轻筛选
6. 新增“我想吃这个”主卡
7. 新增“发给恋人/搭子”操作
8. 新增发起动作表单：
   - 收货人
   - 地址
   - 电话
   - 备注
   - 想说的话
   - 行动方式（请我吃 / 一起去 / 收藏周末）
9. 提交后创建 `eat_share` 动作
10. 动作提交后同步写入记忆预备记录
11. 新增“查看回应状态”
12. 新增“标记已完成”
13. 新增“吃后感受”记录表单
14. 新增一键跳转美团
15. 保留菜谱和相关文章能力，但降级为辅助区
16. 页面视觉统一为更温暖、更清晰的卡片式结构

### 模块 5：关系页重构

目标：把空白 `love` 页面升级成真正的“关系空间”。

步骤：

1. 新建关系页数据结构
2. 无关系时显示邀请绑定引导
3. 支持选择关系类型
4. 支持输入手机号/备注/纪念日
5. 关系建立后展示：
   - 关系头像
   - 关系昵称
   - 绑定天数
   - 本周互动数
   - 最近待回应动作
   - 最近完成记录
6. 新增“今日想一起做什么”快捷入口
7. 新增“吃过 / 去过 / 拍过”回忆入口
8. 新增“解除关系”安全确认
9. 页面样式与首页统一

### 模块 6：发布与发现模板化

目标：让帖子服务主线，而不是和主线平行。

步骤：

1. 扩展帖子模型字段：
   - `title`
   - `templateType`
   - `summary`
   - `sourceModule`
   - `relationId`
   - `actionId`
   - `memoryId`
   - `status`
2. 调整发布页表单结构
3. 增加模板类型选择：
   - 吃过了
   - 去过了
   - 拍到了
   - 完成了
   - 被回应了
4. 从吃模块跳转发布时自动带入模板数据
5. 发布页增加“这条记录是否同步到关系回忆”
6. 后端创建帖子接口兼容新字段
7. 发现页兼容展示新字段
8. 发现页优先展示更结构化的标题和摘要

### 模块 7：记忆与仪表盘

目标：让完成的事情被记录下来，形成长期价值。

步骤：

1. 新增记忆记录写入逻辑
2. 完成动作时自动沉淀回忆
3. 首页展示最近 3 条回忆
4. 关系页展示最近时间线
5. 为后续周报/月报保留统计字段

### 模块 8：联调与兜底

目标：保证关键链路能跑，缺少配置时也能继续开发。

步骤：

1. 检查首页接口联调
2. 检查关系邀请链路
3. 检查吃模块发起动作链路
4. 检查动作回应链路
5. 检查完成后回忆沉淀链路
6. 检查模板化发布链路
7. 检查发现页展示兼容性
8. 缺少 `CLOSEAI_API_KEY` 时保留兜底文案
9. 缺少腾讯云 COS 配置时保留默认媒体上传失败提示
10. 汇总需要你后续补充的配置项

## 三、本轮优先级

### P0

1. 关系模型与接口
2. 首页仪表盘
3. 吃模块发起动作
4. 关系页基础版
5. 模板化发布基础版

### P1

1. 记忆沉淀
2. 首页最近回忆
3. 吃后记录
4. 发现页模板化兼容

### P2

1. 玩模块接入
2. 拍模块接入
3. 通知和订阅消息增强

## 四、本轮交付原则

1. 先让主线真实可跑，再做华丽扩展
2. 先接后端关系链路，再做前端精修
3. 已有 `closeAI` 和腾讯云 COS 能力优先复用，不重复造轮子
4. 如果遇到必须依赖密钥的部分，先保留接口和兜底，不阻塞整体推进
